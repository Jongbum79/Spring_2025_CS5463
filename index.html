Large Language Models (LLMs) and Large Video Models (LVMs) require a significant amount of computational power for inference. It has been proven that as model size and the number of iterations increase, model performance improves, which means that the computational cost required to inference the model continues to rise. There are various approaches to reduce the required computational power like parallelization, inference friendly model design, model quantization and model minimization.  I plan to research and categorize various approaches to problem solving, then analyze the latest algorithms for each approach and summarize the related issues.

 

l  Bibliography of literature (March 5)

 

1.     Algorithm Improvement

A.     FlashAttention (TrioDao et al., 2022 NeurIPS)

                                                i.           Proposes an IO-aware attention mechanism to reduce memory access costs significantly. Achieves up to 3x speedup on GPUs by optimizing attention computation. Fewer passes and efficient tiling improve overall performance.

B.      Switch Transformers (William Fedus et al., 2022 JMLR)

                                                i.           Introduces a mixture-of-experts approach for efficient model scaling. Routes tokens to specialized subnetworks, reducing compute demands. Demonstrates notable training efficiency gains over dense models.

C.      Speculative Sampling (Charlie et al., 2023 OpenAI)

                                                i.           Explores a sampling method to accelerate inference by predicting token sequences. Reduces computation while maintaining output quality effectively. Shows practical latency improvements in real-world applications.

D.     Retentive Network (Yutao Sun et al., 2023 ICLR24)

                                                i.           Presents a retention-based architecture for efficient sequential modeling. Minimizes memory usage while maintaining competitive performance levels. Offers a scalable solution for long-context tasks.

E.      DeepSpeed-MoE (Samyam Rajbhandari et al., 2022 ICML)

                                                i.           Enhances mixture-of-experts training with DeepSpeed optimizations. Reduces memory and compute requirements significantly. Achieves high speedup and scalability on large clusters.

 

2.     GPU/Distributed Parallelism

A.     ZeRo-Offload & ZeRo-Infinity (Jie Ren et al., 2021 MSR)

                                                i.           Enables large model training by offloading data to CPU/NVMe. Minimizes GPU memory constraints without sacrificing performance. Supports billion-parameter models seamlessly.

B.      Alpa (lianmin et al., 2022 OSDI)

                                                i.           Automates parallelism strategies for distributed training efficiency. Optimizes computation and memory across heterogeneous clusters. Simplifies scaling for large-scale model deployments.

C.      Colossal-AI (Shenggui Li et al., ICPP 2022)

                                                i.           Provides a framework for distributed training with advanced parallelism. Reduces memory footprint while maintaining high throughput. Supports efficient scaling for massive models.

 

3.     Model Compression

A.     SparseGPT (Elias Frantar et al., 2023, ICML)

                                                i.           Introduces a pruning method for GPT models with high sparsity. Achieves minimal accuracy loss in one-shot pruning. Speeds up inference for large-scale applications.

B.      Sheared LLaMA (Xiao et al., 2023, arXiv)

                                                i.           Proposes a pruned version of LLaMA for resource-constrained environments. Balances efficiency and performance through effective compression. Targets practical deployment scenarios.

C.      MiniLLM (Yuxian Gu et al., 2024, ICLR)

                                                i.           Develops compact LLMs using knowledge distillation techniques. Maintains quality with a smaller footprint for deployment. Focuses on resource-limited settings effectively.

D.     LoRA and Low-Rank Compression (Hu et al., 2022, ICLR)

                                                i.           ntroduces low-rank adaptation for efficient fine-tuning. Reduces parameter updates while preserving model performance. Enables scalable training with minimal overhead.

E.      AMD: Automatic Multi-step Distillation (Zhuang et al., 2024, ECCV)

                                                i.           Automates multi-step knowledge distillation for LLM efficiency. Improves resource usage through iterative refinement. Achieves competitive accuracy with reduced compute needs.

 

4.     Quantization and Mixed Precision

A.     GPTQ (Elias Frantar et al., 2023, ICLR)

                                                i.           Presents a quantization technique for GPT models at 4-bit precision. Maintains accuracy while significantly speeding up inference. Reduces memory demands for large models.

B.      LLM.int8() (Tim Dettmers et al., 2022, NeurIPS)

                                                i.           Enables 8-bit integer inference for LLMs on low-memory hardware. Optimizes memory usage without compromising performance. Broadens accessibility for resource-limited devices.

C.      ZeroQuant (Zhewei Yao et al., 2022, NeurIPS)

                                                i.           Proposes a quantization framework for LLMs with mixed precision. Minimizes accuracy loss while enhancing inference speed. Targets efficient GPU-based deployments.

D.     SmoothQuant (Xiao et al., 2022, ICML Workshop)

                                                i.           Introduces quantization by smoothing activation outliers effectively. Improves low-bit performance with minimal degradation. Enhances model efficiency for practical use.

E.      AWQ (Ji Lin et al., 2023, MLSys)

                                                i.           Activation-aware weight quantization reduces model size significantly. Preserves accuracy while enabling faster inference. Targets efficient deployment on various hardware.

 

5.     Memory Optimization

A.     ZeRO-Inference (Ramineni et al., 2022, DeepSpeed Blog)

                                                i.           Optimizes inference by offloading parameters to reduce GPU memory needs. Supports large-scale models on limited hardware setups. Ensures minimal latency impact during inference.

B.      FlexGen (Ying Sheng et al., 2023, arXiv)

                                                i.           Enables high-throughput inference on single GPUs with memory-efficient strategies. Focuses on offloading and compression for accessibility. Improves performance on constrained hardware.

C.      vLLM and PagedAttention (Woosuk Kwon et al., 2023, SOSP)

                                                i.           Introduces virtualized LLMs with paged attention for better memory management. Boosts inference throughput via efficient key-value caching. Scales well for large contexts.

 

6.     Optimization for Mobile Devices

A.     LLMCad (Mengwei Xu et al., 2023, arXiv)

                                                i.           Optimizes LLMs for mobile devices using adaptive computation offloading. Reduces latency and power usage on edge hardware. Enhances mobile inference efficiency.

B.      EdgeShard (Peiyuan Zhang et al., 2024, arXiv)

                                                i.           Proposes model sharding for edge devices to distribute computation. Improves inference speed on resource-constrained mobile platforms. Enables efficient mobile deployment.

C.      AWQ & TinyChat (Song Han et al., 2023, MLSys)

                                                i.           Combines AWQ with TinyChat for lightweight LLM inference on mobile. Achieves real-time performance with reduced memory demands. Optimizes for resource-limited environments.

D.     Llama.cpp (Georgi Gerganov, 2023)

                                                i.           Implements efficient LLM inference in C++ for minimal hardware. Optimizes for speed and low resource usage across platforms. Provides a lightweight solution for deployment.

E.      TinyLlama (Peiyuan Zhang et al., 2024, arXiv)

                                                i.           Develops a compact LLaMA variant tailored for mobile use. Retains strong performance with a drastically reduced model size. Focuses on efficiency for edge devices.
